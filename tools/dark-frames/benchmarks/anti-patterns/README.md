# Dark Room · Anti-Patterns

> Piezas (internas o externas) que demuestran qué NO funciona. Calibración inversa para visual-reviewer + concept-reviewer.

## Cómo registrar un anti-pattern

1. Crear archivo `<concept-id>-analysis.md` con el análisis estructurado (qué falla + lección).
2. Si la pieza es interna (output Dark Room) y <50MB, copiar a esta carpeta como referencia.
3. Si la pieza es externa, dejar solo .md con `source_url`.
4. Update `../BENCHMARKS-INDEX.md` añadiendo entrada con resumen 1 línea.

## Cuándo añadir un anti-pattern

- Pieza Dark Room publicada con engagement <50% del promedio cuenta
- Pieza Dark Room que Pablo descarta como "no WOW" después de revisar
- Pieza externa que es ejemplo claro de qué NO hacer (con permiso de citar)
