### 2.2 Paginas de Servicio: Estructura tipo

Cada pagina de servicio (`/servicios/[servicio]/`) debe contener:

1. **H1** con keyword transaccional principal
2. **Parrafo introductorio** (100-150 palabras) con keyword en las primeras 100 palabras
3. **Seccion "Que incluye"** — lista de entregables concretos
4. **Seccion "Como trabajamos"** — proceso paso a paso (genera confianza)
5. **Seccion "Tecnologias"** — stack que usamos (Next.js, Supabase, etc.)
6. **Seccion "Precios orientativos"** o "Desde X euros" (responde a intent de precio)
7. **Casos de exito** relacionados (2-3 con imagen y resultado medible)
8. **FAQ** con schema markup (5-8 preguntas frecuentes)
9. **CTA** claro: "Solicita presupuesto sin compromiso"
10. **Internal links** a soluciones, sectores y articulos de blog relacionados

### 2.3 Paginas Locales: Estructura tipo

Cada pagina local (`/en/[ciudad]/`) debe contener:

1. **H1:** "Agencia Digital en [Ciudad] — Diseno Web, Apps y Marketing"
2. **Contenido unico** sobre el mercado digital en esa ciudad (minimo 800 palabras)
3. **Lista de servicios** disponibles en esa ciudad con links internos
4. **Testimonios** de clientes de esa zona (si existen)
5. **Mencion de zonas/barrios** especificos para reforzar relevancia local
6. **Google Maps embed** (si hay oficina fisica o coworking)
7. **Schema LocalBusiness** con datos NAP (Name, Address, Phone)
8. **CTA** localizado: "Hablemos sobre tu proyecto en [Ciudad]"

### 3.4 Formato de Articulo Optimizado

Cada articulo debe seguir esta estructura:

```
- Title tag: [Keyword principal] — PACAME (max 60 caracteres)
- Meta description: Resumen con CTA (max 155 caracteres)
- H1: Titulo unico con keyword (puede diferir del title tag)
- Intro: 100-150 palabras, keyword en las primeras 100 palabras
- Tabla de contenidos: Con links ancla
- H2/H3: Subtitulos con keywords secundarias y long-tail
- Contenido: Minimo 1.500 palabras (pillar pages: 3.000+)
- Imagenes: Minimo 3, con alt text descriptivo con keyword
- CTA interno: Link a pagina de servicio o presupuesto
- FAQ: 3-5 preguntas con schema FAQ markup
- Autor: "Equipo PACAME" con schema Article/author
- Fecha: Visible y actualizada (schema dateModified)
```

## 4. SEO TECNICO

### 4.1 Checklist de Implementacion Tecnica

#### Meta Tags

- [ ] **Title tag** unico en cada pagina (max 60 caracteres, keyword al inicio)
- [ ] **Meta description** unica en cada pagina (max 155 caracteres, con CTA)
- [ ] **Canonical tag** en todas las paginas para evitar contenido duplicado
- [ ] **Hreflang** solo si se crea version en ingles (no prioritario fase 1)
- [ ] **Open Graph tags** para compartir en redes (og:title, og:description, og:image)
- [ ] **Twitter Card tags** (twitter:card, twitter:title, twitter:description, twitter:image)
- [ ] **Robots meta** (index,follow en paginas publicas; noindex en paginas internas/admin)
- [ ] **Viewport meta** para responsive (`width=device-width, initial-scale=1`)

#### Schema Markup (Datos Estructurados)

Implementar con JSON-LD en el `<head>`:

- [ ] **Organization**: Nombre, logo, redes sociales, fundador
```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "PACAME",
  "description": "Agencia digital espanola con equipo de agentes IA especializados",
  "url": "https://pacame.es",
  "logo": "https://pacame.es/logo.png",
  "founder": {
    "@type": "Person",
    "name": "Pablo Calleja"
  },
  "sameAs": [
    "https://www.instagram.com/pacame.es",
    "https://www.linkedin.com/company/pacame",
    "https://www.tiktok.com/@pacame.es"
  ]
}
```

- [ ] **LocalBusiness**: Para SEO local (en cada pagina de ciudad)
```json
{
  "@context": "https://schema.org",
  "@type": "ProfessionalService",
  "name": "PACAME — Agencia Digital",
  "address": {
    "@type": "PostalAddress",
    "addressLocality": "[Ciudad]",
    "addressCountry": "ES"
  },
  "areaServed": {
    "@type": "Country",
    "name": "Espana"
  },
  "serviceType": ["Diseno Web", "Marketing Digital", "Desarrollo Software", "SEO", "Branding"]
}
```

- [ ] **FAQPage**: En paginas de servicio y articulos con FAQ
- [ ] **Article**: En cada articulo de blog (headline, author, datePublished, dateModified, image)
- [ ] **BreadcrumbList**: En todas las paginas con navegacion jerarquica
- [ ] **Service**: En cada pagina de servicio individual
- [ ] **Review/AggregateRating**: Cuando tengamos resenas de clientes
- [ ] **HowTo**: En articulos tipo guia/tutorial
- [ ] **WebSite** con SearchAction: Para aparecer con barra de busqueda en SERPs

#### Sitemap XML

- [ ] Generar sitemap.xml automaticamente (Next.js tiene soporte nativo)
- [ ] Incluir todas las paginas indexables
- [ ] Excluir paginas noindex, paginas de admin, paginas de error
- [ ] Incluir `<lastmod>` con fecha real de ultima modificacion
- [ ] Incluir `<changefreq>` y `<priority>` segun tipo de pagina:
  - Home: priority 1.0, changefreq weekly
  - Servicios: priority 0.9, changefreq monthly
  - Blog: priority 0.7, changefreq weekly
  - Ciudades: priority 0.6, changefreq monthly
- [ ] Subir sitemap a Google Search Console
- [ ] Subir sitemap a Bing Webmaster Tools

#### Core Web Vitals

Objetivos (todos en "Good"):

| Metrica | Objetivo | Como conseguirlo |
|---------|----------|-----------------|
| **LCP** (Largest Contentful Paint) | < 2.5s | Optimizar imagen hero, preload fuentes, SSR con Next.js |
| **INP** (Interaction to Next Paint) | < 200ms | Minimizar JavaScript, lazy load componentes pesados |
| **CLS** (Cumulative Layout Shift) | < 0.1 | Definir width/height en imagenes, reservar espacio para ads/embeds |

Acciones tecnicas para Core Web Vitals:

- [ ] **Imagenes**: Usar formato WebP/AVIF, lazy loading nativo, next/image con sizes
- [ ] **Fuentes**: Preload fuentes criticas, usar `font-display: swap`, limitar a 2 fuentes
- [ ] **CSS**: Inline CSS critico, purgar CSS no utilizado (Tailwind lo hace automaticamente)
- [ ] **JavaScript**: Code splitting por ruta (Next.js lo hace por defecto), defer scripts no criticos
- [ ] **Server-Side Rendering**: Usar SSR o SSG en todas las paginas publicas (Next.js App Router)
- [ ] **CDN**: Vercel Edge Network o Cloudflare para servir assets globalmente
- [ ] **Compresion**: Activar Brotli/Gzip en el servidor
- [ ] **Prefetch**: Usar next/link con prefetch para navegacion rapida

#### Rastreo e Indexacion

- [ ] Asegurar que todas las paginas importantes son accesibles en maximo 3 clicks desde home
- [ ] No usar JavaScript para navegacion critica (Next.js renderiza HTML por defecto)
- [ ] Implementar paginacion correcta en blog (rel="next"/"prev" o scroll infinito con SSR)
- [ ] Configurar Google Search Console y verificar propiedad
- [ ] Configurar Bing Webmaster Tools
- [ ] Solicitar indexacion manual de paginas nuevas via Search Console (URL Inspection)
- [ ] Monitorizar errores de rastreo semanalmente

#### Seguridad y Rendimiento

- [ ] HTTPS obligatorio en todo el sitio (certificado SSL)
- [ ] Redireccion 301 de http:// a https:// y de www a no-www (o viceversa)
- [ ] Configurar headers de seguridad (HSTS, X-Content-Type, X-Frame-Options)
- [ ] Implementar CSP (Content Security Policy) basico

#### URLs y Redirecciones

- [ ] URLs limpias, en minuscula, con guiones, sin parametros innecesarios
- [ ] Maximo 3 niveles de profundidad (pacame.es/nivel1/nivel2/nivel3)
- [ ] Redireccion 301 para cualquier URL que cambie
- [ ] Pagina 404 personalizada con links utiles y buscador
- [ ] Evitar cadenas de redirecciones (maximo 1 salto)

- [ ] Estructura correcta de headings (un solo H1, H2/H3 jerarquicos)
