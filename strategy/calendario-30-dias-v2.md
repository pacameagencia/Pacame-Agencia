## 10 · Bot LLM cualificación DMs · voz Pablo

### Funciones del bot

Cuando llega un DM/comentario:

1. **Saludar con tono Pablo** (basado en `IDENTIDAD-PABLO.md` · realismo brutal · tutear · frases cortas · sin humo)
2. **Cualificar en 3 preguntas máximo**:
   - ¿Qué tipo de negocio tienes (o quieres montar)?
   - ¿Qué problema tienes ahora mismo?
   - ¿Qué buscas: web, SaaS llave en mano, acceso a Dark Room, o consulta?
3. **Routing según respuesta**:
   - Web/landing → propone llamada Calendly + envía mini-portfolio
   - SaaS llave en mano → explica oferta 2.000€ + Calendly de descubrimiento
   - Dark Room → envía link `darkroomcreative.cloud` + explica lifetime + sistema referidos
   - Consulta general → responde directo (con voz Pablo) y guarda lead frío en Supabase
4. **Guardar lead** en Supabase tabla `leads_cualificados`:
   - timestamp, ig_user_id, ig_username, mensajes, cualificación, intent, status (frío/tibio/caliente), fecha_próximo_seguimiento
5. **Notificar Pablo Telegram** si lead caliente (ej. cliente menciona presupuesto 2k€+ o "lo necesito ya")

### Implementación técnica

- Endpoint webhook IG ya existe: `web/app/api/instagram/webhook/route.ts` (verificar)
- Handler nuevo:
  - Parsea evento DM
  - Llama a Claude Sonnet o Nebius DeepSeek con system prompt = `IDENTIDAD-PABLO.md` + few-shot examples + cualificación protocol
  - Stores conversation context en Supabase
  - Envía respuesta vía `web/lib/instagram.ts::sendInstagramDM()`
- Cron diario que consolida leads + envía resumen Telegram a Pablo

### Tabla Supabase `leads_cualificados`

```sql
create table leads_cualificados (
  id uuid primary key default gen_random_uuid(),
  ig_user_id text not null,
  ig_username text,
  first_seen_at timestamptz default now(),
  last_message_at timestamptz default now(),
  intent text check (intent in ('web','saas','darkroom','consulta','otro')),
  temperatura text check (temperatura in ('frio','tibio','caliente')),

  conversation jsonb,
  status text default 'open' check (status in ('open','contacted','closed_won','closed_lost')),
```

### Flow

2. Para el slot actual (mañana o tarde):

   - Si no existe asset → genera con script correspondiente (composer + opcional gpt-image-2)
   - Llama `publish-carrusel.mjs <carpeta>`
3. Stories cron (cada 4 horas): identifica story del slot · upload + publica con IG Stories API
4. Logs en Telegram cada publicación · si falla, alerta
